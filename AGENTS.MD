### Step 1 — THINK 🧠
State root cause, blast radius, security implications in one sentence.

### Step 2 — READ 📖
Read the target file(s) from disk. Check recent git history.

### Step 3 — PLAN 📋
Write the exact change: file → function/line → old behavior → new behavior. Describe verification. Flag risks.

### Step 4 — APPROVE ✅
Present the plan to Captain. Wait for explicit go-ahead.

### Step 5 — EXECUTE ⚡
Make only the changes described. Verify it works, report result. If it fails, go back to Step 1.
